# 시간 복잡도 (Big-O 표기법)

## 점근적 분석이란?

점근적 분석은 입력 크기가 무한대로 갈 때 알고리즘의 자원 사용(시간 또는 공간)이 어떻게 증가하는지를 설명합니다. 상수 계수와 하위 차항을 무시하고 **성장률**에 집중합니다.

정확한 연산 횟수를 세지 않습니다. 알고리즘이 어떻게 **확장되는지** 분류합니다.

```
10n² + 500n + 10000
     ↓ (n → ∞)
     O(n²)
```

n이 클 때 n² 항이 지배합니다. 상수(10, 500, 10000)는 규모가 커지면 무관해집니다.

---

## 점근적 표기법

### Big-O (상한)

**O(f(n))**: 알고리즘이 큰 n에서 (상수 계수까지) **최대** f(n) 시간이 걸립니다.

```
f(n) = O(g(n)) ⟺ ∃ c > 0, n₀ : f(n) ≤ c·g(n), ∀ n ≥ n₀
```

Big-O는 **최악의 경우** 상한을 설명합니다.

### Big-Ω (하한)

**Ω(f(n))**: 알고리즘이 **최소** f(n) 시간이 걸립니다.

```
f(n) = Ω(g(n)) ⟺ ∃ c > 0, n₀ : f(n) ≥ c·g(n), ∀ n ≥ n₀
```

### Big-Θ (정확한 한계)

**Θ(f(n))**: 알고리즘이 (상수 계수 내에서) **정확히** f(n) 시간이 걸립니다. O(f(n))이면서 Ω(f(n))입니다.

### 실제 사용

대부분의 사람들은 "Θ(n)"을 의미할 때 "O(n)"이라고 말합니다. 엄밀히:
- "이진 탐색은 O(n)이다" — **참** (O(n²), O(n³) 등도 맞지만) 도움이 안 됨
- "이진 탐색은 Θ(log n)이다" — 정확함
- "이진 탐색은 O(log n)이다" — 보통 이렇게 표현

## 주요 복잡도 클래스

| 복잡도 | 이름 | 예시 |
|---|---|---|
| O(1) | 상수 | 배열 인덱스 접근, 해시 테이블 조회 |
| O(log n) | 로그 | 이진 탐색 |
| O(n) | 선형 | 선형 탐색, 단일 루프 |
| O(n log n) | 선형로그 | 병합 정렬, 퀵 정렬 (평균) |
| O(n²) | 이차 | 중첩 루프, 버블 정렬 |
| O(n³) | 삼차 | 행렬 곱셈 (단순) |
| O(2ⁿ) | 지수 | 재귀 피보나치 (단순), 부분집합 |
| O(n!) | 팩토리얼 | 순열, 브루트포스 TSP |

### 성장률 비교

n = 1,000,000일 때:

| O(1) | O(log n) | O(n) | O(n log n) | O(n²) | O(2ⁿ) |
|---|---|---|---|---|---|
| 1 | 20 | 10⁶ | 2×10⁷ | 10¹² | 10³⁰⁰⁰⁰⁰ |
| 즉시 | 즉시 | 1 ms | 20 ms | 11.5일 | 우주의 열사 |

## 코드 분석 방법

### 규칙 1: 단순 구문은 O(1)

```python
x = 5                    # O(1)
y = x + 10               # O(1)
arr[0] = 42               # O(1)
```

### 규칙 2: 루프

시간 = 반복 횟수 × 본문 비용:

```python
# O(n) — 루프가 n번 실행, 본문 O(1)
for i in range(n):
    print(i)

# O(n²) — 중첩 루프
for i in range(n):
    for j in range(n):
        print(i, j)

# O(n × m) — 다른 루프 범위
for i in range(n):
    for j in range(m):
        print(i, j)
```

### 규칙 3: 반감/배가 루프는 O(log n)

```python
# O(log n) — i가 매번 2배
i = 1
while i < n:
    i *= 2               # log₂(n)번 반복

# O(log n) — 반감
i = n
while i > 1:
    i //= 2              # log₂(n)번 반복
```

### 규칙 4: 순차 → 더하기, 중첩 → 곱하기

```python
# O(n + m) — 순차
for i in range(n):       # O(n)
    process(i)
for j in range(m):       # O(m)
    process(j)

# O(n × m) — 중첩
for i in range(n):       # O(n)
    for j in range(m):   # × O(m)
        process(i, j)
```

### 규칙 5: 상수와 하위 차항 제거

```
3n² + 5n + 100 → O(n²)
2ⁿ + n³ → O(2ⁿ)
n log n + n → O(n log n)
```

### 규칙 6: 재귀 알고리즘

**점화식**을 세우고 풀기:

```python
# T(n) = T(n-1) + O(1) → O(n)
def factorial(n):
    if n <= 1: return 1
    return n * factorial(n - 1)

# T(n) = 2T(n/2) + O(n) → O(n log n)  (병합 정렬)
def merge_sort(arr):
    if len(arr) <= 1: return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

# T(n) = 2T(n-1) + O(1) → O(2ⁿ)
def fibonacci(n):
    if n <= 1: return n
    return fibonacci(n - 1) + fibonacci(n - 2)
```

## 마스터 정리

**T(n) = aT(n/b) + O(nᵈ)** 형태의 점화식에 대해:

| 조건 | 결과 |
|---|---|
| d > log_b(a) | T(n) = O(nᵈ) |
| d = log_b(a) | T(n) = O(nᵈ log n) |
| d < log_b(a) | T(n) = O(n^(log_b(a))) |

예시:
- **이진 탐색**: T(n) = T(n/2) + O(1) → a=1, b=2, d=0 → O(log n)
- **병합 정렬**: T(n) = 2T(n/2) + O(n) → a=2, b=2, d=1 → O(n log n)
- **슈트라센**: T(n) = 7T(n/2) + O(n²) → a=7, b=2, d=2 → O(n^2.81)

## 분할 상환 분석

일부 연산은 가끔 비용이 높지만 평균적으로는 저렴합니다. **분할 상환** 분석은 연산 시퀀스에 대한 평균 비용을 제공합니다.

### 동적 배열 (ArrayList)

동적 배열의 append는 분할 상환 O(1):
- 대부분의 append: O(1) — 요소 배치만
- 가끔 리사이즈: O(n) — 모든 요소를 새 배열로 복사

크기를 2배로 늘릴 경우, n번 append의 총 비용 = n + n/2 + n/4 + ... ≈ 2n → append당 분할 상환 O(1).

### 방법

| 방법 | 아이디어 |
|---|---|
| **집계** | n개 연산의 총 비용 / n |
| **회계** | 저렴한 연산에 더 청구하여 비싼 연산을 "대비" |
| **포텐셜** | 포텐셜 함수 정의, 비용 = 실제 비용 + ΔΦ |

## 공간 복잡도

알고리즘이 (입력 외에) 사용하는 **추가 메모리** 양.

```python
# O(1) 공간 — 상수 추가 메모리
def find_max(arr):
    max_val = arr[0]
    for x in arr:
        if x > max_val:
            max_val = x
    return max_val

# O(n) 공간 — 새 배열 생성
def reverse_copy(arr):
    return arr[::-1]

# O(n) 공간 — 재귀 스택
def factorial(n):
    if n <= 1: return 1
    return n * factorial(n - 1)    # n개 스택 프레임
```

### 재귀 스택 공간

각 재귀 호출은 스택 프레임을 추가합니다. 깊이 d의 재귀는 O(d) 공간 사용:

| 알고리즘 | 재귀 깊이 | 스택 공간 |
|---|---|---|
| 이진 탐색 | O(log n) | O(log n) |
| 병합 정렬 | O(log n) | O(log n) + O(n) 병합 버퍼 |
| 퀵 정렬 | O(log n) 평균, O(n) 최악 | O(log n) 평균 |
| DFS | O(V) | O(V) |
| 피보나치 (단순) | O(n) | O(n) |

## 실무 고려사항

### 상수가 중요한 경우

상수 1000의 O(n)은 상수 1의 O(n log n)보다 n < ~10,000에서 느립니다. Big-O는 상수를 숨깁니다.

실제 성능 영향 요인:
- **캐시 지역성**: 순차적 배열 접근이 랜덤 포인터 추적보다 훨씬 빠름
- **분기 예측**: 예측 가능한 분기가 더 저렴
- **메모리 할당**: 힙 할당은 비용이 높음
- **명령어 수준 병렬성**: 현대 CPU는 사이클당 여러 연산 실행

### 경쟁 프로그래밍 시간 제한 가이드

| n | 최대 복잡도 | 대략적 연산 수 |
|---|---|---|
| ≤ 10 | O(n!) | 순열 가능 |
| ≤ 20-25 | O(2ⁿ) | 부분집합 가능 |
| ≤ 500 | O(n³) | 1.25억 연산 |
| ≤ 5,000 | O(n²) | 2,500만 연산 |
| ≤ 10⁶ | O(n log n) | 2,000만 연산 |
| ≤ 10⁸ | O(n) | 1억 연산 |
| > 10⁸ | O(log n) 또는 O(1) | 이진 탐색, 수학 |

## 복잡도 클래스 (P, NP, NP-완전)

| 클래스 | 설명 | 예시 |
|---|---|---|
| **P** | 다항 시간에 풀 수 있음 | 정렬, 최단 경로, MST |
| **NP** | 다항 시간에 검증 가능 | SAT, TSP, 그래프 색칠 |
| **NP-완전** | NP에서 가장 어려운 문제 | 3-SAT, 정점 커버, 부분합 |
| **NP-난해** | NP-완전 이상으로 어려움 | 정지 문제, 최적화 TSP |

**P = NP?** 문제는 빠르게 검증할 수 있는 모든 문제를 빠르게 풀 수도 있는지 묻습니다. 컴퓨터 과학에서 가장 중요한 미해결 문제 중 하나입니다.

NP-완전 문제에 대한 접근법:
- **근사 알고리즘** — 다항 시간에 근최적해 보장
- **휴리스틱** — 보장은 없지만 실제로 잘 동작
- **백트래킹 + 가지치기** — 정확하지만 최악 지수 시간
- **매개변수화 알고리즘** — 매개변수 k에 지수적, n에 다항적
