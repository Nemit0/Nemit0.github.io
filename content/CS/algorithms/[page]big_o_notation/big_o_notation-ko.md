# 빅오 표기법과 알고리즘 분석

점근적 표기법은 알고리즘이 어떻게 확장되는지를 설명하는 수학적 언어입니다. 기계별 상수를 제거하고 순수하게 성장률에 집중합니다. 이 글에서는 **공식적인 정의**, 경계를 **엄밀하게 증명하고 도출하는 방법**, 그리고 자주 발생하는 **실수들**을 다룹니다.

> **참고:** 이 글은 표기법 자체와 도출 방법에 집중합니다. 주요 복잡도 클래스의 실용적 개요와 코드를 빠르게 분석하는 방법은 [시간 복잡도 글](/CS/algorithms/time_complexity)을 참고하세요.

---

## 1. 공식적 정의가 중요한 이유

"O(n)은 선형"이고 "O(n²)은 중첩 루프"라는 직관만으로도 어느 정도는 통합니다. 하지만 직관은 경계 조건에서 무너집니다:

- $f(n) = 0.001n^2 + 1000n$은 $O(n)$인가, $O(n^2)$인가?
- 모든 $O(n \log n)$ 알고리즘이 모든 $O(n^2)$ 알고리즘보다 빠른가?
- 언제 $T(n) = O(n \log n)$이 아닌 $T(n) = \Theta(n \log n)$이라고 말할 수 있는가?

공식적 정의는 이 질문들에 정확하게 답하고, 경계를 추측이 아닌 *증명*으로 확인할 수 있게 해줍니다.

---

## 2. Big-O: 공식적 정의

**정의.** $f(n) = O(g(n))$이 성립하는 필요충분조건은 양의 상수 $c$와 $n_0$가 존재하여:

$$f(n) \leq c \cdot g(n) \quad \text{for all } n \geq n_0$$

을 만족하는 것입니다.

읽기: "$f$는 결국 $c \cdot g$보다 위로 유계된다."

### 상수의 의미

- **$c$** — 상수 배율. 고정된 오버헤드를 흡수합니다. $f(n) = 100n$이고 $g(n) = n$이면 $c = 100$으로 선택합니다.
- **$n_0$** — "eventually(결국)"의 임계값. $n_0$ 아래에서는 경계가 성립하지 않을 수도 있지만, 그 위에서는 항상 성립합니다. 이를 통해 소규모 입력의 동작을 무시할 수 있습니다.

Big-O는 다음에 대해 **아무것도** 말하지 않습니다:
- 주어진 입력에 대한 정확한 실행 시간
- $g(n)$이 *타이트한* 경계인지 여부 (매우 느슨할 수 있음)
- 작은 $n$에서의 동작

### 정의 시각화

```
        value
          │              c·g(n)
          │           ╱
          │         ╱   ← f(n)이 모든 n ≥ n₀에서
          │       ╱       c·g(n) 아래에 머뭄
          │ ╱───╱
          │╱
          └─────────────────────── n
                   n₀
```

## 3. Big-Ω: 공식적 정의

**정의.** $f(n) = \Omega(g(n))$이 성립하는 필요충분조건은 양의 상수 $c$와 $n_0$가 존재하여:

$$f(n) \geq c \cdot g(n) \quad \text{for all } n \geq n_0$$

을 만족하는 것입니다.

읽기: "$f$는 결국 $c \cdot g$에 의해 *아래로* 유계된다." Big-Ω는 **하한**을 나타냅니다 — 크기 $n$의 모든 입력에서 알고리즘이 달성할 수 있는 최선.

---

## 4. Big-Θ: 공식적 정의

**정의.** $f(n) = \Theta(g(n))$이 성립하는 필요충분조건은:

$$f(n) = O(g(n)) \quad \text{이면서} \quad f(n) = \Omega(g(n))$$

동치로, 양의 상수 $c_1$, $c_2$, $n_0$가 존재하여:

$$c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n) \quad \text{for all } n \geq n_0$$

Big-Θ는 **타이트한 경계** — $f$와 $g$는 상수 계수까지 동일한 비율로 성장합니다. 이것이 가장 정밀한 점근적 진술입니다.

---

## 5. Little-o와 Little-ω

이들은 Big-O와 Big-Ω의 *엄격한* 버전으로 — 경계가 타이트하지 않음을 나타냅니다.

### Little-o

**정의.** $f(n) = o(g(n))$이 성립하는 필요충분조건은:

$$\lim_{n \to \infty} \frac{f(n)}{g(n)} = 0$$

비공식적으로: $f$는 $g$보다 *엄격하게 느리게* 성장합니다. 아무리 작은 $\epsilon > 0$에 대해서도, 결국에는 $f(n) < \epsilon \cdot g(n)$이 됩니다.

**예시:**
- $n = o(n^2)$ — 선형은 이차보다 엄격하게 느림
- $\log n = o(n)$ — 로그는 선형보다 엄격하게 느림
- $n^2 \neq o(n^2)$ — 동일한 비율로 성장하므로 성립하지 *않음*

### Little-ω

**정의.** $f(n) = \omega(g(n))$이 성립하는 필요충분조건은:

$$\lim_{n \to \infty} \frac{f(n)}{g(n)} = \infty$$

비공식적으로: $f$는 $g$보다 *엄격하게 빠르게* 성장합니다.

**예시:**
- $n^2 = \omega(n)$
- $2^n = \omega(n^{100})$ — 지수는 임의의 다항식을 지배

### 요약 표

| 표기법 | 의미 | 비유 |
|---|---|---|
| $f = O(g)$ | $f$는 많아야 $g$만큼 빠르게 성장 | $f \leq g$ |
| $f = \Omega(g)$ | $f$는 최소한 $g$만큼 빠르게 성장 | $f \geq g$ |
| $f = \Theta(g)$ | $f$는 정확히 $g$만큼 빠르게 성장 | $f = g$ |
| $f = o(g)$ | $f$는 $g$보다 엄격하게 느리게 성장 | $f < g$ |
| $f = \omega(g)$ | $f$는 $g$보다 엄격하게 빠르게 성장 | $f > g$ |

## 6. 경계 증명: 단계별

$f(n) = O(g(n))$을 증명한다는 것은 정의를 만족하는 $c$와 $n_0$의 **명시적 값을 찾는** 것을 의미합니다. 마법이 아니라 그냥 대수학입니다.

### 증명 전략

1. $f(n)$을 작성한다
2. 각 항을 $g(n)$만 포함하는 것으로 상한 처리
3. 경계 처리 단계가 유효하도록 충분히 큰 $n_0$를 선택
4. 결과적인 $c$를 명시적으로 지정

---

### 예시 1: $3n^2 + 5n + 7 = O(n^2)$ 증명

**주장:** $f(n) = 3n^2 + 5n + 7 = O(n^2)$

**증명:**

$n \geq 1$에서:

$$5n \leq 5n^2 \quad \text{($n \geq 1$이면 $n \leq n^2$이므로)}$$
$$7 \leq 7n^2 \quad \text{($n \geq 1$이면 $1 \leq n^2$이므로)}$$

따라서:

$$3n^2 + 5n + 7 \leq 3n^2 + 5n^2 + 7n^2 = 15n^2$$

$c = 15$, $n_0 = 1$을 찾았으며, 모든 $n \geq 1$에서 $f(n) \leq 15 \cdot n^2$이 성립합니다. $\blacksquare$

> **핵심 기법:** 모든 저차항을 $\text{(계수)} \cdot n^d$로 대체합니다. 여기서 $d$는 최고차항의 차수입니다.

---

### 예시 2: $\frac{1}{2}n^2 - 3n = \Omega(n^2)$ 증명

**주장:** $f(n) = \frac{1}{2}n^2 - 3n = \Omega(n^2)$

**증명:**

어떤 $c > 0$에 대해 $f(n) \geq c \cdot n^2$임을 보여야 합니다.

$$\frac{1}{2}n^2 - 3n \geq c \cdot n^2$$

양변을 $n^2$으로 나누면 ($n > 0$일 때 유효):

$$\frac{1}{2} - \frac{3}{n} \geq c$$

$n \geq 12$에서: $\frac{3}{n} \leq \frac{3}{12} = \frac{1}{4}$이므로, $\frac{1}{2} - \frac{3}{n} \geq \frac{1}{4}$.

$c = \frac{1}{4}$, $n_0 = 12$를 선택하면, 모든 $n \geq 12$에서 $f(n) \geq \frac{1}{4} n^2$. $\blacksquare$

---

### 예시 3: $\frac{1}{2}n^2 - 3n = \Theta(n^2)$ 증명

**주장:** $f(n) = \frac{1}{2}n^2 - 3n = \Theta(n^2)$

**증명:**

*상한* (예시 1의 기법): $n \geq 1$에서, $3n \leq 3n^2$이므로:

$$\frac{1}{2}n^2 - 3n \leq \frac{1}{2}n^2 \leq n^2$$

따라서 $n \geq 1$에서 $f(n) \leq 1 \cdot n^2$ → $c_2 = 1$로 $f(n) = O(n^2)$.

*하한*: 예시 2에서, $n \geq 12$에서 $f(n) \geq \frac{1}{4} n^2$.

$n_0 = 12$, $c_1 = \frac{1}{4}$, $c_2 = 1$로 결합하면:

$$\frac{1}{4} n^2 \leq \frac{1}{2}n^2 - 3n \leq n^2 \quad \text{모든 } n \geq 12\text{에서}$$

따라서 $f(n) = \Theta(n^2)$. $\blacksquare$

---

### 예시 4: $2^n \neq O(n^{100})$ 증명

**주장:** $2^n$은 $O(n^{100})$이 아닙니다.

**모순에 의한 증명:** $2^n = O(n^{100})$이라 가정합니다. 그러면 모든 $n \geq n_0$에서 $2^n \leq c \cdot n^{100}$을 만족하는 $c, n_0$가 존재합니다. 하지만:

$$\frac{2^n}{n^{100}} \to \infty \quad \text{as } n \to \infty$$

(지수는 임의의 다항식보다 빠르게 성장 — L'Hôpital 또는 귀납법으로 확인). 따라서 어떤 상수 $c$도 큰 $n$에서 $2^n / n^{100}$을 위에서 유계할 수 없습니다. 모순. $\blacksquare$

이것은 동치로 $n^{100} = o(2^n)$으로 쓸 수 있습니다.

## 7. 유용한 속성과 규칙

### 전이성

$f = O(g)$이고 $g = O(h)$이면, $f = O(h)$. ($\Omega$와 $\Theta$에도 동일하게 적용.)

### 합 규칙 (순차적 구성)

$$O(f) + O(g) = O(\max(f, g))$$

```python
def example(n):
    # 1단계: O(n)
    for i in range(n):
        print(i)

    # 2단계: O(n^2)
    for i in range(n):
        for j in range(n):
            print(i, j)

# 전체: O(n) + O(n²) = O(n²)
```

지배적인 항이 이깁니다. 저차 단계는 흡수됩니다.

### 곱 규칙 (중첩 구성)

$$O(f) \cdot O(g) = O(f \cdot g)$$

```python
def example(n):
    for i in range(n):        # O(n) 반복
        for j in range(n):    # 각각 O(n) 반복
            print(i + j)      # 반복당 O(1)

# 전체: O(n) * O(n) * O(1) = O(n²)
```

### 다항식 규칙

임의의 다항식 $p(n) = a_k n^k + a_{k-1} n^{k-1} + \ldots + a_0$에 대해:

$$p(n) = \Theta(n^k)$$

최고차항만 중요합니다.

### 로그 밑 변환

$$\log_a n = \frac{\log_b n}{\log_b a} = \Theta(\log_b n)$$

모든 로그는 상수 계수만큼만 다르므로, **점근적 표기법에서 밑은 중요하지 않습니다**. 항상 $O(\log n)$으로 씁니다.

### 상수 배수

$$O(c \cdot f(n)) = O(f(n)) \quad \text{임의의 상수 } c > 0\text{에 대해}$$

이것은 정의의 직접적인 결과입니다: 스칼라 배율은 항상 Big-O 정의(2절)의 상수 $c$에 흡수될 수 있으므로 클래스에 영향을 주지 않습니다.

## 8. 코드에서 Big-O 도출

### 8.1 단일 루프

```python
def linear_search(arr, target):
    for i in range(len(arr)):   # n번 반복
        if arr[i] == target:    # 반복당 O(1)
            return i
    return -1
```

**도출:** 루프는 최대 $n$번 실행됩니다. 각 반복은 $O(1)$ 작업(배열 접근 및 비교)을 수행합니다. 곱 규칙에 의해: $n \cdot O(1) = O(n)$.

---

### 8.2 중첩 루프 — 종속적 경계

```python
def print_pairs(n):
    for i in range(n):          # 외부: n번 반복
        for j in range(i + 1, n):   # 내부: n-1-i번 반복
            print(i, j)
```

**도출:** 내부 루프의 총 반복 횟수를 계산합니다:

$$\sum_{i=0}^{n-1} (n - 1 - i) = \sum_{k=0}^{n-1} k = \frac{n(n-1)}{2}$$

다항식 규칙에 의해, $\frac{n(n-1)}{2} = \Theta(n^2)$이므로 함수는 $O(n^2)$입니다.

> 흔한 혼동: 모든 중첩 루프가 $O(n^2)$인 것은 아닙니다. 내부 경계를 신중하게 확인해야 합니다.

---

### 8.3 중첩 루프 — 독립적 경계

```python
def matrix_add(A, B, n, m):
    for i in range(n):
        for j in range(m):
            A[i][j] += B[i][j]
```

**도출:** 외부 루프는 $n$번, 내부는 $m$번, 본문은 $O(1)$.

$$T(n, m) = n \cdot m \cdot O(1) = O(nm)$$

$m = n$일 때, $O(n^2)$으로 단순화됩니다.

---

### 8.4 로그 루프

```python
def count_halvings(n):
    count = 0
    i = n
    while i > 1:     # 매 반복마다 절반으로 줄어듦
        i //= 2
        count += 1
    return count
```

**도출:** $k$번 반복 후, $i = \lfloor n / 2^k \rfloor$. 루프는 $i \leq 1$, 즉 $n / 2^k \leq 1$, 즉 $k \geq \log_2 n$일 때 종료됩니다. 루프는 $\lfloor \log_2 n \rfloor + 1 = O(\log n)$번 실행됩니다.

---

### 8.5 O(n log n) 인식

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])   # T(n/2)
    right = merge_sort(arr[mid:])  # T(n/2)
    return merge(left, right)      # O(n)
```

**점화식으로 도출:** 점화식은:

$$T(n) = 2T(n/2) + O(n)$$

재귀 트리 전개:

```
레벨 0:  크기 n의 문제 1개  →  비용: n
레벨 1:  크기 n/2의 문제 2개  →  비용: 2 × n/2 = n
레벨 2:  크기 n/4의 문제 4개  →  비용: 4 × n/4 = n
...
레벨 k:  크기 n/2^k의 문제 2^k개  →  비용: n
...
레벨 log₂n:  크기 1의 문제 n개  →  비용: n
```

$\log_2 n + 1$개의 레벨이 있으며, 각 레벨은 $O(n)$ 작업을 기여합니다. 합계: $O(n \log n)$.

**마스터 정리**로도 풀 수 있습니다 (아래 참조).

## 9. 마스터 정리

$a \geq 1$, $b > 1$인 형태 $T(n) = aT(n/b) + f(n)$의 점화식에 대해:

$d = \log_b a$라 하면.

| 경우 | 조건 | 결과 |
|---|---|---|
| 경우 1 | $f(n) = O(n^{d - \epsilon})$ (어떤 $\epsilon > 0$에 대해) | $T(n) = \Theta(n^d)$ |
| 경우 2 | $f(n) = \Theta(n^d)$ | $T(n) = \Theta(n^d \log n)$ |
| 경우 3 | $f(n) = \Omega(n^{d + \epsilon})$ (어떤 $\epsilon > 0$에 대해, 정규성 조건 포함) | $T(n) = \Theta(f(n))$ |

### 마스터 정리 적용

**병합 정렬:** $T(n) = 2T(n/2) + O(n)$

- $a = 2$, $b = 2$, $d = \log_2 2 = 1$
- $f(n) = O(n) = O(n^1) = \Theta(n^d)$ → 경우 2
- 결과: $T(n) = \Theta(n \log n)$ ✓

**이진 탐색:** $T(n) = T(n/2) + O(1)$

- $a = 1$, $b = 2$, $d = \log_2 1 = 0$
- $f(n) = O(1) = \Theta(n^0) = \Theta(n^d)$ → 경우 2
- 결과: $T(n) = \Theta(\log n)$ ✓

**단순 행렬 곱셈:** $T(n) = 8T(n/2) + O(n^2)$

- $a = 8$, $b = 2$, $d = \log_2 8 = 3$
- $f(n) = O(n^2) = O(n^{3 - 1})$ → 경우 1 ($\epsilon = 1$)
- 결과: $T(n) = \Theta(n^3)$ ✓

```

시각적 기울기 (로그 척도):

      │

    ╱
n log n ─╱
        ╱
n    ──╱
      ╱╱╱
log n ──────────────
1    ════════════════  (수평)
```

**핵심 요점:** 대규모에서 $O(n^2)$와 $O(n \log n)$ 사이의 격차는 엄청납니다. $n = 10^6$ 입력에서 $O(n^2)$ 알고리즘은 약 $10^{12}$번 연산(며칠 분량의 계산)을 수행하는 반면, $O(n \log n)$은 $2 \times 10^7$번(밀리초 단위)입니다.

---

## 11. 흔한 실수와 함정

### 실수 1: O(2n) ≠ O(n)으로 생각하기

```python
# 이것은 O(2n)이 아닌 O(n)입니다:
def two_passes(arr):
    total = 0
    for x in arr:   # 첫 번째 패스
        total += x
    for x in arr:   # 두 번째 패스
        print(x)
    return total
```

$O(2n) = O(n)$입니다. 왜냐하면 스칼라 상수는 Big-O 정의(2절)의 상수 $c$에 흡수되기 때문입니다. 2는 단순히 $c$의 일부가 되며, 경계는 여전히 성립합니다. $n$개 요소에 대한 두 개의 순차적 루프는 $O(2n)$이 아닌 $O(n)$입니다.

---

### 실수 2: O와 Θ 혼동

"버블 정렬은 O(n)이다"라고 말하는 것은 **기술적으로 맞지만** 쓸모가 없습니다 — 버블 정렬은 $O(n^{1000})$, $O(2^n)$ 등이기도 합니다. Big-O는 상한이므로, 실제 성장률보다 위에 있는 모든 것이 유효합니다.

| 말하는 것 | 의미 | 맞나요? |
|---|---|---|
| 버블 정렬은 $O(n^2)$ | 맞는 상한 | ✓ 정확 |
| 버블 정렬은 $\Theta(n^2)$ | 타이트한 경계 | ✓ 정밀 |
| 버블 정렬은 $O(n)$ | 맞지만 오해 소지 | 기술적으로 유효, 실용적으로 틀림 |
| 버블 정렬은 $O(n^3)$ | 맞는 상한 | ✓ 하지만 느슨함 |

"이진 탐색은 $O(\log n)$이다"라고 말할 때 보통 $\Theta(\log n)$을 의미합니다. 타이트한 경계를 의미할 때 $\Theta$를 사용하는 습관을 기르세요.

---

### 실수 3: 지배적이지 않은 항을 너무 일찍 제거하기

```python
def process(n, m):
    for i in range(n):       # O(n)
        pass
    for i in range(m * m):   # O(m²)
        pass
```

$n$과 $m$이 **독립 변수**라면, 복잡도는 $O(m^2)$가 아닌 $O(n + m^2)$입니다. $n = O(m^2)$임을 알 때만 $n$을 제거할 수 있습니다.

---

### 실수 4: 중첩 루프가 항상 O(n²)인 것은 아님

```python
# 이것은 O(n²)이 아닌 O(n log n)입니다:
def halving_inner(n):
    for i in range(n):   # O(n) 외부
        j = n
        while j > 0:     # O(log n) 내부
            j //= 2
```

내부 루프는 매번 절반으로 줄어들므로, `i`에 관계없이 $O(\log n)$번 실행됩니다. 합계: $O(n \log n)$.

---

### 실수 5: 입력 종속적 동작 무시

```python
def search_and_process(arr, target):
    for i, x in enumerate(arr):   # 최대 O(n)
        if x == target:
            # ... O(1) 작업
            return i
    return -1
```

**최악의 경우**는 $O(n)$ (목표를 찾지 못함). **최선의 경우**는 $O(1)$ (목표가 첫 번째 요소). **평균의 경우** (균등 분포)는 $O(n/2) = O(n)$.

Big-O 자체는 어떤 경우가 적용되는지 말해주지 않으므로, 명시적으로 표현하세요:
- 최악의 경우: $O(n)$
- 최선의 경우: $\Omega(1)$
- 평균의 경우: $\Theta(n)$

---

### 실수 6: O(log n)이 이진 탐색만을 의미한다고 가정하기

$O(\log n)$은 문제 크기가 각 단계에서 *상수 비율로 나뉘는* 모든 곳에서 나타납니다:
- 이진 탐색 (2로 나눔)
- $n$의 자릿수 세기 (10으로 나눔)
- 균형 이진 트리 순회 (높이 $= \log_2 n$)
- 제곱에 의한 거듭제곱 (`pow(x, n)`이 $O(\log n)$)

```python
def fast_power(base, exp):
    if exp == 0:
        return 1
    if exp % 2 == 0:
        half = fast_power(base, exp // 2)  # 문제가 절반으로 줄어듦
        return half * half
    return base * fast_power(base, exp - 1)
```

각 재귀 호출은 (짝수일 때) `exp`를 절반으로 줄이므로, 이것은 $O(\log n)$번의 곱셈입니다.

---

### 실수 7: 분할상환 비용을 최악의 경우로 처리하기

Python의 `list.append()`는 **분할상환** $O(1)$ — 가끔 $O(n)$ 크기 재조정을 트리거합니다. 단일 호출 최악의 경우는 $O(n)$이지만, $n$번의 append 전체에서 총 비용은 $O(n)$이므로, 분할상환 비용은 append당 $O(1)$입니다.

```python
arr = []
for i in range(n):
    arr.append(i)   # 분할상환 O(1) — 전체 루프는 O(n²)이 아닌 O(n)
```

연산당 최악의 경우를 사용하면 $O(n^2)$으로 과대 계산되어 잘못된 결과가 나옵니다.

---

## 12. 처음부터 분석하는 예시

### 예시 A: 이 함수의 복잡도 구하기

```python
def mystery(n):
    result = 0
    i = 1
    while i <= n:         # (1)
        j = 0
        while j < i:      # (2)
            result += 1
            j += 1
        i *= 2            # (3)
    return result
```

**1단계:** 외부 루프 (1)은 몇 번 실행되나요?  
`i`는 $1, 2, 4, 8, \ldots, 2^k$의 값을 취하며 $2^k > n$이 될 때까지 계속되므로, $k = \lfloor \log_2 n \rfloor$. 외부 루프는 $O(\log n)$번 실행됩니다.

**2단계:** 외부 반복당 내부 루프 (2)는 몇 번 실행되나요?  
`i = 2^k`일 때, 내부 루프는 $2^k$번 실행됩니다.

**3단계:** 총 내부 반복 횟수:

$$\sum_{k=0}^{\lfloor \log_2 n \rfloor} 2^k = 2^{\lfloor \log_2 n \rfloor + 1} - 1 \leq 2n - 1$$

**결론:** $T(n) = O(n)$. $O(n \log n)$이나 $O(\log n)$이 아닙니다 — 외부 인덱스의 두 배 증가가 내부 합을 $O(n)$으로 수렴하게 합니다.

---

### 예시 B: 행렬 거듭제곱 (단순)

```python
def mat_pow(M, n, size):
    """행렬 M을 n승합니다."""
    result = identity_matrix(size)       # O(size²)
    for _ in range(n):                   # n번 반복
        result = mat_multiply(result, M, size)  # 각각 O(size³)
    return result
```

**도출:**
- `identity_matrix`: $O(s^2)$, 여기서 $s =$ `size`
- 루프: $n$번 반복
- `mat_multiply`: 호출당 $O(s^3)$ (단순 3중 중첩 루프)
- 합계: $O(s^2) + n \cdot O(s^3) = O(n \cdot s^3)$

$n$과 $s$ 모두 변수라면, 복잡도는 $O(n s^3)$입니다. $s$가 고정되면 (예: $4 \times 4$ 회전 행렬), $O(n)$으로 단순화됩니다.

---

### 예시 C: 루프 내 문자열 구성

```python
# 버전 1 — 단순 연결
def build_string_v1(n):
    s = ""
    for i in range(n):
        s += str(i)    # Python에서 연결당 O(len(s))!
    return s

# 버전 2 — join
def build_string_v2(n):
    parts = []
    for i in range(n):
        parts.append(str(i))   # 분할상환 O(1)
    return "".join(parts)      # O(총 길이) = O(n log n) — 자릿수 변동
```

**버전 1 분석:**  
반복 $i$에서, `s`의 길이는 대략 $i \cdot \lceil \log_{10} i \rceil$입니다. 연결은 전체 문자열을 복사합니다:

$$T(n) = \sum_{i=0}^{n-1} O(i \log i) = O(n^2 \log n)$$

**버전 2 분석:**  
`append`는 분할상환 $O(1)$이므로 루프는 $O(n)$입니다. `join`은 모든 문자를 한 번 복사하여, $O(\text{총 문자}) = O(n \log n)$ ($0..n$ 숫자의 자릿수 합이 $O(n \log n)$). 합계: $O(n \log n)$.

> 루프 내 단순 문자열 연결은 고전적인 $O(n^2)$ 함정입니다.

## 13. 표기법들 간의 관계

자주 마주치는 복잡도 클래스의 계층 구조 (가장 느리게 성장하는 것에서 가장 빠른 것 순):

$$O(1) \subset O(\log n) \subset O(\sqrt{n}) \subset O(n) \subset O(n \log n) \subset O(n^2) \subset O(n^3) \subset O(2^n) \subset O(n!)$$

여기서 $\subset$은 "~의 진부분집합" — 모든 $O(1)$ 함수는 $O(\log n)$, $O(n)$ 등이기도 합니다. Big-O는 위로 누적됩니다.

해당하는 $\Theta$ 클래스는 **서로소** 입니다: 함수는 정확히 하나의 $\Theta$ 클래스에 속합니다. 함수의 $\Theta$ 클래스를 알면 정확한 성장률을 알 수 있습니다.

## 14. 빠른 참고

### $f = O(g)$ 증명

1. $f(n)$을 작성한다
2. 저차항의 상한 처리: 각 $a \cdot n^k$를 지배항의 차수 $d$를 이용해 $a \cdot n^d$로 대체
3. 상수를 합산 → $c$ 결정
4. 2단계의 모든 대체가 유효한 $n_0 \geq 1$을 찾는다

### $f \neq O(g)$ 증명

$\lim_{n \to \infty} f(n)/g(n) = \infty$임을 보인다.

### 코드에서 도출

| 코드 패턴 | 복잡도 |
|---|---|
| 단일 문장 | $O(1)$ |
| $O(1)$ 본문의 $n$번 루프 | $O(n)$ |
| 각 단계에서 절반으로 줄어드는 루프 | $O(\log n)$ |
| $n$의 두 개의 순차적 루프 | $O(n)$ |
| 둘 다 $n$인 중첩 루프 | $O(n^2)$ |
| $n$ 루프, 내부는 절반으로 | $O(n \log n)$ |
| 분할 정복, 절반으로 나눔 | 마스터 정리 적용 |
| 점화식 $T(n) = 2T(n/2) + O(n)$ | $O(n \log n)$ |
| 점화식 $T(n) = T(n-1) + O(1)$ | $O(n)$ |
| 점화식 $T(n) = 2T(n-1) + O(1)$ | $O(2^n)$ |

## 요약

- **Big-O** ($O$)는 상한: 알고리즘이 *최대* 이만큼 자원을 사용합니다.
- **Big-Ω** ($\Omega$)는 하한: 알고리즘이 *최소* 이만큼 자원을 사용합니다.
- **Big-Θ** ($\Theta$)는 타이트한 경계: 양쪽 경계가 일치 — 가장 정밀한 진술.
- **Little-o** ($o$)와 **little-ω** ($\omega$)는 *엄격한* (비타이트) 순서를 표현합니다.
- 경계를 *증명*하려면, 정의를 만족하는 명시적 상수 $c$와 $n_0$를 찾으세요.
- 상수와 저차항은 항상 제거됩니다 — 하지만 어떤 항이 지배적인지 확인한 *후에만*.
- 누군가 일상적 대화에서 "$O(f(n))$"이라고 말할 때, 보통 $\Theta(f(n))$을 의미합니다. 공식적인 맥락에서는 정밀하게 사용하세요.
